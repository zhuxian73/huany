## 三、改进项明细

### 1. 数据模型重构

**问题**：`conversations` 和 `messages` 是两个互不关联的状态，切换会话时消息不会跟随切换。

**方案**：将消息内嵌到会话中，统一数据结构。

```javascript
// 重构前
conversations = [{ id, title }]
messages = [全局消息数组]  // 不关联会话

// 重构后
conversations = [{
  id,
  title,
  modelId,        // 每个会话记录使用的模型
  createdAt,
  updatedAt,
  messages: [{ id, role, content, tokens, images, replyTo, createdAt }]
}]
```

**影响文件**：`hooks/useConversations.js`

### 3. 切换会话加载对应消息

**问题**：点击侧边栏会话只更新 `currentConversationId`，右侧消息不切换。

**方案**：从当前会话派生消息列表，切换时自动停止生成并清除回复目标。

```javascript
const currentConversation = conversations.find(c => c.id === currentConversationId)
const currentMessages = currentConversation?.messages ?? []
```

**影响文件**：`hooks/useConversations.js`、`components/ChatArea.jsx`

---

### 4. 首条消息后自动生成会话标题

**问题**：新建会话标题永远是"新对话"。

**方案**：发送首条消息时，根据内容截取前 20 字符生成标题。

```javascript
export const generateTitle = (text) => {
  const trimmed = text.trim().replace(/[#*`>\-\n]/g, ' ').replace(/\s+/g, ' ').trim()
  if (!trimmed) return '新对话'
  return trimmed.length > 20 ? trimmed.slice(0, 20) + '…' : trimmed
}
```

**影响文件**：`utils/simulateReply.js`、`hooks/useConversations.js`

### 5. 修复流式输出内存泄漏

**问题**：`setInterval` 和 `setTimeout` 在组件卸载或切换会话时不会被清理，导致内存泄漏。

**方案**：用 `useRef` 保存定时器 ID，在 `useEffect` cleanup 中清除。

```javascript
const intervalRef = useRef(null)
const timeoutRef = useRef(null)

useEffect(() => {
  return () => {
    if (intervalRef.current) clearInterval(intervalRef.current)
    if (timeoutRef.current) clearTimeout(timeoutRef.current)
  }
}, [])
```

切换会话时也调用 `stopGenerating()` 清理定时器。

**影响文件**：`hooks/useConversations.js`

### 7. XSS 防护

**问题**：`dangerouslySetInnerHTML` 直接渲染 `marked.parse()` 输出，用户输入 `<script>` 会被执行。

**方案**：引入 `DOMPurify`，对 Markdown 渲染结果进行净化。

```javascript
import DOMPurify from 'dompurify'

<div dangerouslySetInnerHTML={{ __html: DOMPurify.sanitize(marked.parse(msg.content)) }} />
```

**影响文件**：`components/MessageRow.jsx`、`package.json`（新增 `dompurify` 依赖）

### 9. 错误处理与加载态

**问题**：无错误边界，`clipboard.writeText` 无错误处理，模型加载无 loading 状态。

**方案**：

- **ErrorBoundary**：class 组件捕获子组件渲染异常，显示友好错误页
- **clipboard**：`async` 调用 + `try/catch`
- **模型加载**：`isLoadingModels` 状态 + 下拉菜单 disabled

```javascript
const copyToClipboard = async (text) => {
  try {
    await navigator.clipboard.writeText(text)
  } catch (err) {
    console.error('复制失败:', err)
  }
}
```

**影响文件**：`components/ErrorBoundary.jsx`、`components/MessageRow.jsx`、`components/ModelSelector.jsx`
